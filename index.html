계정 상태

Google 또는 Kakao 계정을 연결해도 기존 아기 기록은 그대로 유지됩니다.

OAuth 계정은 접근 권한 계층일 뿐이며, records의 실제 owner는 family workspace입니다.

같은 계정으로 다른 기기에 로그인하면 기존 가족 기록을 자동으로 불러옵니다. 다른 보호자 계정에서 함께 사용하려면 가족 연결 코드를 입력해야 합니다.

현재 익명 사용자로 사용 중입니다. 카카오 계정을 연결하면 다른 기기에서도 가족 기록을 이어서 사용할 수 있습니다.

Kakao/Google 로그인이 작동하려면 Supabase Dashboard에서 Provider와 Redirect URL 설정이 필요합니다.

같은 계정 기기 복원 상태

같은 Google/Kakao 계정으로 로그인하면 family_members 기준으로 가족 공간과 기록을 복원합니다.

가족 공유 테스트 방법

1. 이 기기에서 기록을 1~2개 생성합니다.

2. 다른 기기 또는 다른 브라우저에서 Google/Kakao 계정으로 로그인합니다.

3. 같은 계정이면 기존 가족 기록이 자동으로 복원되어야 합니다.

4. 다른 보호자 계정이면 가족 연결 코드를 입력해야 같은 기록을 볼 수 있습니다.

5. 가족 공유에는 계정 코드가 아니라 가족 연결 코드를 사용하세요.

가족/아기 서버 구조

가족/아기 구조는 Phase 4 로그인 기능으로 넘어가기 위한 준비 단계예요. 현재는 익명 사용자 기준으로 안전하게 연결해 둡니다.

가족/아기 구조를 확인하면 현재 서버 연결 상태를 볼 수 있어요.

가족 연결

가족 연결 코드를 사용하면 다른 보호자 계정이나 새 기기에서 같은 family workspace에 연결하고 서버 기록을 이 기기에 복원할 수 있어요. 계정 코드는 가족 공유에 사용하지 않습니다.

가족 초대 링크

가족 초대 링크는 가족 연결 코드와 같은 민감도를 가집니다. 가까운 보호자에게만 공유하세요.

가족 연결 코드가 준비되면 초대 링크가 표시됩니다.

링크를 받은 사람은 확인 버튼을 눌러야 가족 workspace에 참여합니다.

연결 전 현재 로컬 기록과 설정을 자동 백업합니다.

가족 초대 링크를 통해 접속했습니다. 이 가족 workspace에 참여하면 같은 아기 기록을 함께 사용할 수 있습니다. 참여하시겠습니까?

가족/아기 서버 구조를 확인하면 가족 연결 코드와 연결된 기기 목록을 볼 수 있어요.

가족 구성원

현재 이 가족 기록을 함께 사용하는 계정을 보여줍니다. 구성원 제거는 다음 버전에서 지원 예정입니다.

가족 구성원 정보를 아직 불러오지 않았습니다.

계정 연결/전환 진단

계정을 연결하거나 전환해도 현재 family_id를 유지하는 것이 목표입니다. 연결 후 기록이 보이지 않으면 가족 초대 링크 또는 가족 연결 코드를 사용해 복구할 수 있습니다.

provider 전환 기록이 아직 없습니다.

서버에 저장된 기록을 불러와 이 기기의 기록과 비교할 수 있어요. 병합 전에는 현재 기기 데이터가 자동 백업됩니다.

서버 기록을 불러오면 병합 미리보기를 만들 수 있어요.

현재는 익명 사용자로 서버 저장을 준비하고 있어요. 나중에 로그인 기능이 추가되면 여러 기기에서 더 안전하게 사용할 수 있어요.
